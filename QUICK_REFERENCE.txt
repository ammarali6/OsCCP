FIVE SCHEDULING ALGORITHMS
═══════════════════════════════════════════════════════════════════════════

┌─────────────────────────────────────────────────────────────────────────┐
│ 1️⃣  FCFS - First Come First Served                                      │
│    • Simple, fair scheduling                                            │
│    • Processes execute in arrival order                                 │
│    • Non-preemptive                                                     │
│    • Prone to convoy effect                                             │
│    • Best for: Batch processing with uniform times                      │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ 2️⃣  SJF - Shortest Job First                                            │
│    • Optimal average waiting time                                       │
│    • Executes shortest burst time process first                         │
│    • Non-preemptive (Preemptive version available: SRTF)               │
│    • Risk of starvation                                                 │
│    • Best for: Short jobs with known burst times                        │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ 3️⃣  Priority Scheduling                                                 │
│    • Executes highest priority process first                            │
│    • Lower priority number = higher priority                            │
│    • Non-preemptive                                                     │
│    • Risk of starvation (low-priority processes)                        │
│    • Best for: Real-time systems with priorities                        │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ 4️⃣  Round Robin                                                         │
│    • Fair scheduling with time quantum                                  │
│    • Each process gets fixed time slice                                 │
│    • Preemptive                                                         │
│    • Good responsiveness                                                │
│    • Best for: Time-sharing, interactive systems                        │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ 5️⃣  MLFQ - Multi-Level Feedback Queue                                   │
│    • Adaptive scheduling with 3 queues                                  │
│    • Moves processes between queues                                     │
│    • Prevents starvation                                                │
│    • Flexible and sophisticated                                         │
│    • Best for: Mixed workloads, unknown job lengths                     │
└─────────────────────────────────────────────────────────────────────────┘


METRICS EXPLAINED
═══════════════════════════════════════════════════════════════════════════

⏱️  Waiting Time
    Formula: Start Time - Arrival Time
    Ideal:   Low (processes don't wait long)
    Impact:  User satisfaction

📈 Turnaround Time
    Formula: Finish Time - Arrival Time
    Ideal:   Low (quick process completion)
    Impact:  System throughput

💻 CPU Utilization
    Formula: (Busy Time / Total Time) × 100%
    Range:   0-100%
    Ideal:   High (>90% is good)

🚀 Throughput
    Formula: Number of Processes / Total Time
    Ideal:   High (more processes per unit time)
    Impact:  System capacity

WHEN TO USE EACH ALGORITHM
═══════════════════════════════════════════════════════════════════════════

FCFS
    ✓ Batch processing
    ✓ Uniform job times
    ✓ Non-critical systems
    ✗ Not for real-time

SJF
    ✓ Known short job times
    ✓ Minimize waiting time
    ✓ Laboratory experiments
    ✗ Starvation risk

Priority
    ✓ Mixed priority workloads
    ✓ Real-time systems
    ✓ System processes (high priority)
    ✗ Starvation of low-priority

Round Robin
    ✓ Time-sharing systems
    ✓ Interactive applications
    ✓ Fair scheduling needed
    ✗ Higher overhead

MLFQ
    ✓ Mixed workloads
    ✓ Unknown job lengths
    ✓ Responsive I/O systems
    ✗ Complex, many parameters

STATISTICAL INTERPRETATION
═══════════════════════════════════════════════════════════════════════════

High Waiting Time:
    → Algorithm schedules processes inefficiently
    → Users experience long delays
    → Consider SJF or Round Robin

High Turnaround Time:
    → Processes take too long to complete
    → May indicate long burst times or queue wait
    → Consider algorithm with better scheduling

Low CPU Utilization:
    → CPU sitting idle too much
    → Processes arriving sparsely
    → Increase process frequency

Low Throughput:
    → Few processes finishing per time unit
    → Processes taking long time or algorithm inefficient
    → Consider faster algorithm
